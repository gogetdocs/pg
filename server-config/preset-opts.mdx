## PostgreSQL Preset Options

The following preset options are read-only and report various aspects of PostgreSQL behavior that might be of interest to certain applications, particularly administrative front-ends.

### Block Size

The `block_size` option reports the size of a disk block. It is determined by the value of `BLCKSZ` when building the server. The default value is 8192 bytes.

**SQL Example:**

```sql
SHOW block_size;
```

**Output:**

```sql
 block_size
------------
       8192
```

### Data Checksums

The `data_checksums` option reports whether data checksums are enabled for this cluster.

**SQL Example:**

```sql
SHOW data_checksums;
```

**Output:**

```sql
 data_checksums
----------------
 on
```

### Data Directory Mode

The `data_directory_mode` option reports the permissions the data directory had at server startup.

**SQL Example:**

```sql
SHOW data_directory_mode;
```

**Output:**

```sql
---------------------
               0700
```

### Locale Settings

The `lc_collate` and `lc_ctype` options report the locale in which sorting of textual data is done and the locale that determines character classifications, respectively.

**SQL Example:**

```sql
SHOW lc_collate;
SHOW lc_ctype;
```

**Output:**

```sql
 lc_collate
-------------
 en_US.UTF-8

 lc_ctype
-----------
 en_US.UTF-8
```
