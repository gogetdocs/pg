## PostgreSQL Connection and Authentication Settings

### 20.3.1. Connection Settings

#### **listen_addresses**

Specifies the TCP/IP address(es) on which the server is to listen for connections from client applications. The value takes the form of a comma-separated list of host names and/or numeric IP addresses.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET listen_addresses TO 'localhost, 192.168.1.100';
  ```

#### **port**

The TCP port the server listens on; 5432 by default.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET port TO 5433;
  ```

#### **max_connections**

Determines the maximum number of concurrent connections to the database server.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET max_connections TO 200;
  ```

#### **superuser_reserved_connections**

Determines the number of connection “slots” that are reserved for connections by PostgreSQL superusers.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET superuser_reserved_connections TO 5;
  ```

#### **unix_socket_directories**

Specifies the directory of the Unix-domain socket(s) on which the server is to listen for connections from client applications.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET unix_socket_directories TO '/tmp, /var/run/postgresql';
  ```

#### **unix_socket_group**

Sets the owning group of the Unix-domain socket(s).

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET unix_socket_group TO 'postgres';
  ```

#### **unix_socket_permissions**

Sets the access permissions of the Unix-domain socket(s).

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET unix_socket_permissions TO '0770';
  ```

#### **bonjour**

Enables advertising the server's existence via Bonjour.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET bonjour TO on;
  ```

#### **bonjour_name**

Specifies the Bonjour service name.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET bonjour_name TO 'My PostgreSQL Server';
  ```

#### **tcp_keepalives_idle**

Specifies the amount of time with no network activity after which the operating system should send a TCP keepalive message to the client.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET tcp_keepalives_idle TO '1h';
  ```

#### **tcp_keepalives_interval**

Specifies the amount of time after which a TCP keepalive message that has not been acknowledged by the client should be retransmitted.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET tcp_keepalives_interval TO '10s';
  ```

#### **tcp_keepalives_count**

Specifies the number of TCP keepalive messages that can be lost before the server's connection to the client is considered dead.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET tcp_keepalives_count TO 5;
  ```

#### **tcp_user_timeout**

Specifies the amount of time that transmitted data may remain unacknowledged before the TCP connection is forcibly closed.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET tcp_user_timeout TO '30s';
  ```

#### **client_connection_check_interval**

Sets the time interval between optional checks that the client is still connected, while running queries.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET client_connection_check_interval TO '1s';
  ```

### 20.3.2. Authentication

#### **authentication_timeout**

Maximum amount of time allowed to complete client authentication.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET authentication_timeout TO '30s';
  ```

#### **password_encryption**

Determines the algorithm to use to encrypt the password when a password is specified in `CREATE ROLE` or `ALTER ROLE`.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET password_encryption TO 'scram-sha-256';
  ```

#### **krb_server_keyfile**

Sets the location of the server's Kerberos key file.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET krb_server_keyfile TO '/etc/krb5.keytab';
  ```

#### **krb_caseins_users**

Sets whether GSSAPI user names should be treated case-insensitively.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET krb_caseins_users TO on;
  ```

#### **db_user_namespace**

Enables per-database user names.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET db_user_namespace TO on;
  ```

### 20.3.3. SSL

#### **ssl**

Enables SSL connections.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl TO on;
  ```

#### **ssl_ca_file**

Specifies the name of the file containing the SSL server certificate authority (CA).

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_ca_file TO 'root.crt';
  ```

#### **ssl_cert_file**

Specifies the name of the file containing the SSL server certificate.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_cert_file TO 'server.crt';
  ```

#### **ssl_crl_file**

Specifies the name of the file containing the SSL client certificate revocation list (CRL).

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_crl_file TO 'crl.pem';
  ```

#### **ssl_crl_dir**

Specifies the name of the directory containing the SSL client certificate revocation list (CRL).

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_crl_dir TO '/etc/ssl/crl';
  ```

#### **ssl_key_file**

Specifies the name of the file containing the SSL server private key.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_key_file TO 'server.key';
  ```

#### **ssl_ciphers**

Specifies a list of SSL cipher suites that are allowed to be used by SSL connections.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_ciphers TO 'HIGH:MEDIUM:+3DES:!aNULL';
  ```

#### **ssl_prefer_server_ciphers**

Specifies whether to use the server's SSL cipher preferences, rather than the client's.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_prefer_server_ciphers TO on;
  ```

#### **ssl_ecdh_curve**

Specifies the name of the curve to use in ECDH key exchange.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_ecdh_curve TO 'prime256v1';
  ```

#### **ssl_min_protocol_version**

Sets the minimum SSL/TLS protocol version to use.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_min_protocol_version TO 'TLSv1.2';
  ```

#### **ssl_max_protocol_version**

Sets the maximum SSL/TLS protocol version to use.

- **Example Usage:**
  ```sql
  ALTER SYSTEM SET ssl_max_protocol_version TO 'TLSv1
  ```
