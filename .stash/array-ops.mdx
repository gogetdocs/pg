In PostgreSQL, there are several other operators for working with arrays. Here's a list of common operators you can use with arrays, along with their meanings:

### Array Operators in PostgreSQL:

1. **Contains (`@>`)**:

   - **Syntax**: `array1 @> array2`
   - **Description**: True if `array1` contains all the elements of `array2` (i.e., `array1` is a superset of `array2`).
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] @> ARRAY[1, 2]; -- Returns true
     ```

2. **Contained by (`<@`)**:

   - **Syntax**: `array1 <@ array2`
   - **Description**: True if `array1` is contained within `array2` (i.e., `array2` is a superset of `array1`).
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2] <@ ARRAY[1, 2, 3]; -- Returns true
     ```

3. **Overlap (`&&`)**:

   - **Syntax**: `array1 && array2`
   - **Description**: True if `array1` and `array2` have any elements in common (i.e., they overlap).
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] && ARRAY[2, 4]; -- Returns true
     ```

4. **Equality (`=`)**:

   - **Syntax**: `array1 = array2`
   - **Description**: True if `array1` and `array2` are exactly the same.
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] = ARRAY[1, 2, 3]; -- Returns true
     ```

5. **Inequality (`<>`)**:

   - **Syntax**: `array1 <> array2`
   - **Description**: True if `array1` and `array2` are not the same.
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] <> ARRAY[1, 3, 2]; -- Returns true
     ```

6. **Less Than (`<`)** and **Greater Than (`>`)**:

   - **Syntax**: `array1 < array2` or `array1 > array2`
   - **Description**: These operators compare arrays lexicographically (based on the first differing elements).
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] < ARRAY[1, 3, 4]; -- Returns true
     ```

7. **Less Than or Equal (`<=`)** and **Greater Than or Equal (`>=`)**:
   - **Syntax**: `array1 <= array2` or `array1 >= array2`
   - **Description**: Similar to `<` and `>`, but allow for equality.
   - **Example**:
     ```sql
     SELECT ARRAY[1, 2, 3] <= ARRAY[1, 2, 3]; -- Returns true
     ```

### Array Functions

In addition to operators, PostgreSQL also provides some useful functions for working with arrays:

1. **array_length**:

   - **Description**: Returns the length of the array.
   - **Syntax**: `array_length(array, dimension)`
   - **Example**:
     ```sql
     SELECT array_length(ARRAY[1, 2, 3], 1); -- Returns 3
     ```

2. **unnest**:

   - **Description**: Expands an array to a set of rows (useful for turning an array into a list of values).
   - **Syntax**: `unnest(array)`
   - **Example**:
     ```sql
     SELECT unnest(ARRAY[1, 2, 3]); -- Returns 1, 2, 3 in separate rows
     ```

3. **array_append**:

   - **Description**: Appends an element to the end of an array.
   - **Syntax**: `array_append(array, element)`
   - **Example**:
     ```sql
     SELECT array_append(ARRAY[1, 2], 3); -- Returns ARRAY[1, 2, 3]
     ```

4. **array_prepend**:

   - **Description**: Adds an element to the beginning of an array.
   - **Syntax**: `array_prepend(element, array)`
   - **Example**:
     ```sql
     SELECT array_prepend(0, ARRAY[1, 2, 3]); -- Returns ARRAY[0, 1, 2, 3]
     ```

5. **array_remove**:

   - **Description**: Removes all occurrences of an element from an array.
   - **Syntax**: `array_remove(array, element)`
   - **Example**:
     ```sql
     SELECT array_remove(ARRAY[1, 2, 3, 2], 2); -- Returns ARRAY[1, 3]
     ```

6. **array_position**:

   - **Description**: Returns the position of the first occurrence of an element in an array.
   - **Syntax**: `array_position(array, element)`
   - **Example**:
     ```sql
     SELECT array_position(ARRAY[1, 2, 3], 2); -- Returns 2
     ```

7. **array_agg**:
   - **Description**: Aggregates multiple rows into an array (useful in `GROUP BY` queries).
   - **Syntax**: `array_agg(expression)`
   - **Example**:
     ```sql
     SELECT array_agg(column_name) FROM table_name;
     ```
